# Parallel Browser Execution Artifacts Enhancements

This document summarizes the enhancements made to capture and display artifacts from parallel browser execution agents.

## Overview

The enhancements focus on improving the parallel browser execution agents to capture various artifacts (GIFs, screenshots with element highlights, HAR files, traces) and display them in a user-friendly way in the frontend.

## Backend Enhancements

### 1. Browser Execution Service (`browser_execution_service.py`)

#### Key Changes:
- **Fixed session_id scope issue**: The `session_id` variable is now properly passed to the `_run_agent_in_tab` method
- **Improved artifact path handling**: Artifact paths are now correctly formatted and validated
- **Added file existence checks**: The service now verifies if generated artifacts (like GIFs) actually exist before including them in the results

#### Artifact Organization:
- Each agent gets its own directory for artifacts:
  - Videos: `./recordings/videos/{session_id}/agent_{agent_id}/`
  - Traces: `./recordings/debug.traces/{session_id}/agent_{agent_id}/`
  - Network traces: `./recordings/network.traces/{session_id}/agent_{agent_id}.har`

#### Artifact Generation:
- GIFs are generated by the `TrackingBrowserAgent` when `generate_gif=True`
- HAR files are generated when `record_har_path` is specified
- Debug traces are generated when `traces_dir` is specified

### 2. Tracking Browser Agent (`browser_execution_agent.py`)

#### Key Features:
- **Element interaction tracking**: Tracks clicks and text inputs with comprehensive element details
- **Selector generation**: Generates framework-specific selectors for automation scripts
- **Artifact generation**: Configured to generate GIFs, screenshots, and network traces
- **Element highlighting**: Highlights elements in screenshots for better visualization

## Frontend Enhancements

### 1. Parallel Execution Visualizer (`ParallelExecutionVisualizer.tsx`)

#### Key Improvements:
- **Enhanced artifact preview function**: Now correctly handles different artifact types and generates appropriate API endpoints
- **Improved artifact display**: Shows GIFs, HAR files, and directory information in a structured way
- **Better API endpoint generation**: Dynamically generates correct API endpoints based on artifact paths

#### Artifact Display Features:
- **GIF previews**: Shows animated GIFs directly in the UI
- **HAR file links**: Provides download links for network traces
- **Directory information**: Shows paths to video and trace directories
- **Agent-specific organization**: Groups artifacts by agent/tab for easy navigation

## API Enhancements

### 1. Artifacts API (`artifacts.py`)

#### Endpoints:
- `/api/v1/artifacts/{task_id}/{artifact_type}/{scenario_id}/{filename}` - For complex paths
- `/api/v1/artifacts/{task_id}/{artifact_type}/{filename}` - For simple paths

#### Supported Artifact Types:
- `videos` - GIFs and video recordings
- `network.traces` - HAR files
- `debug.traces` - Debug trace files

## Artifact Types Generated

### 1. GIFs
- Animated recordings of the entire execution
- Generated per agent in `./recordings/videos/{session_id}/agent_{agent_id}/execution.gif`

### 2. HAR Files
- Network traces capturing all HTTP requests/responses
- Generated per agent in `./recordings/network.traces/{session_id}/agent_{agent_id}.har`

### 3. Debug Traces
- Detailed execution traces for debugging
- Generated per agent in `./recordings/debug.traces/{session_id}/agent_{agent_id}/`

### 4. Screenshots
- Element-highlighted screenshots captured during execution
- Part of the browser state information

## Usage Example

When executing parallel browser tests, the system now:

1. Creates unique directories for each agent's artifacts
2. Configures the `TrackingBrowserAgent` with appropriate recording parameters
3. Executes all agents in parallel tabs
4. Captures artifacts during execution
5. Returns structured results with artifact paths
6. Displays artifacts in the frontend UI with proper previews and links

1. **Improved Debugging**: Visual artifacts help developers understand execution flow
2. **Better Test Analysis**: GIFs and screenshots provide visual confirmation of test execution
3. **Enhanced Reporting**: Network traces and debug information aid in troubleshooting
4. **User-Friendly Interface**: Clean organization and display of artifacts in the frontend
